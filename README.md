# Daily Practice Problems - Web & Backend Development

This repository contains my daily practice projects and exercises, organized into clear technical domains as part of the Wipro Great Learning Training.

## Repository Structure

The workspace is divided into three main sections based on the technology stack:

### 1. ☕ Java
Contains core Java and Web/Servlet practice projects:
* **Demo1**: A foundational Java project.
* **JavaEclipse**: Contains Servlet and Interface practice code (e.g., `payment.java`, `HelloServlet.java`, `LoginServlet.java`).

### 2. 🌱 SpringBoot
*Reserved for future Spring Boot microservices and backend API projects.*

### 3. ⚛️ React
Contains all front-end web development practices using React, Vite, and standard HTML/CSS.

#### Core React Concepts
* **ContextAPI**: Demonstrates the use of the React Context API to manage global state without prop drilling.
* **Reducer**: Showcases complex state management using the `useReducer` hook.
* **Modal**: Practice using React's `useMemo` hook to optimize expensive calculations.
* **Portal**: Demonstrates the use of React Portals to render a Modal component outside of the main DOM hierarchy.

#### Form Handling and Validation
* **FormikYup**: Implementsform handling and validation using the Formik library alongside Yup schema validation.
* **ProFormik**: Extended practice for deeply understanding Formik form states, field binding, and error message rendering.
* **reactValidation**: Introductory validation techniques using standard React state.

#### Projects and Applications
* **TodoApp**: A fully functional Todo application interacting with a local API backend (json-server), featuring CRUD and search/filtering capabilities.
* **SimpleIntrestCalculator**: An application calculating Simple Interest from user inputs.
* **sma**: A "Student Management App" managing students and courses using the Context API to share state across sibling components.
* **OddEven**: Utility checker evaluating a numeric state to render whether it is odd or even.

#### HTML and Routing Practices
* **HTML**: Static HTML files testing layout concepts, external stylesheets, and web structures.
* **Practice27 / Practice27.2**: Specific practice modules focusing on component rendering (arrays of objects, student lists, numerical tables) and establishing multi-page routing using `react-router-dom`.
* **ReactJs**: The baseline Vite + React starter template used for bootstrapping most of the projects in this repository.
