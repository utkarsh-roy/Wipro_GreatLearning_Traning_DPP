# Daily Practice Problems - React & Web Development

This repository contains my daily practice projects and exercises for React and web development, specifically as part of the Wipro Great Learning Training.

## Project Structure & Descriptions

Here is a breakdown of all the project folders in this repository, designed to practice various front-end web development skills:

### Core React Concepts
* **ContextAPI**: Demonstrates the use of the React Context API to manage global state without prop drilling. Setup includes a global username provider consumed by nested components.
* **Reducer**: Showcases complex state management using the `useReducer` hook. Implements a simple counter with increment, decrement, and reset actions.
* **Modal**: Practice using React's `useMemo` hook to optimize expensive calculations, combined with basic state for managing count and numbers. Includes a conceptual modal structure.
* **Portal**: Demonstrates the use of React Portals to render a Modal component outside of the main DOM hierarchy, useful for overlays and dialogs.

### Form Handling and Validation
* **FormikYup**: Implementsform handling and validation using the Formik library alongside Yup schema validation. The form handles product name, price, category, and quantity inputs.
* **ProFormik**: Similar to the above, an extended practice for deeply understanding Formik form states, field binding, and error message rendering.
* **reactValidation**: Contains introductory validation techniques using standard React state to capture and validate user input prior to form submission.

### Projects and Applications
* **TodoApp**: A fully functional Todo application interacting with a local API backend (json-server). It includes CRUD operations (Create, Read, Update, Delete), along with dynamic search and filtering capabilities by Name and ID.
* **SimpleIntrestCalculator**: A self-contained application that takes Principal, Rate, and Time inputs the calculates and displays the Simple Interest.
* **sma**: A "Student Management App" demonstrating component composition, managing students and courses using the Context API to share state across sibling components.
* **OddEven**: A quick utility checker that evaluates a numeric state and renders whether it is an odd or even number.

### HTML and Routing Practices
* **HTML**: Contains standard static HTML files testing layout concepts, external stylesheets (`style.css`), and basic web structures.
* **Practice27 / Practice27.2**: Specific practice modules focusing on component rendering (arrays of objects, student lists, numerical tables) and establishing multi-page routing using `react-router-dom` (Home, Profile, Contact).
* **ReactJs**: The baseline Vite + React starter template used for bootstrapping most of the projects in this repository.
